## Card Battle Web Application

Web application inspired by a Pokémon-style card collection and battle system, developed as a fullstack project using Angular and Laravel.

## 🚀 Tech Stack

Frontend: Angular, HTML, CSS, Bootstrap, TypeScript

Backend: Laravel (PHP)

Database: MySQL

Architecture: REST API (client-server)

## 🧠 Features

User authentication (registration and login)

Card collection system

Pack opening with random card generation

Points system

Data persistence using a relational database

Communication between frontend and backend via API

## 🏗️ Architecture

The application follows a client-server architecture:

The Angular frontend handles the user interface and HTTP requests

The Laravel backend exposes a REST API and manages business logic

MySQL is responsible for data storage and persistence

Backend (Laravel)
cd PokemonBattleBack
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate
php artisan serve
Frontend (Angular)
cd PokemonBattle
npm install
ng serve

# Aplicación Web de Batalla de Cartas

Aplicación web inspirada en un sistema de colección y batalla de cartas tipo Pokémon, desarrollada como proyecto fullstack utilizando Angular y Laravel.

## 🚀 Tecnologías utilizadas

* Frontend: Angular, HTML, CSS, Bootstrap, TypeScript
* Backend: Laravel (PHP)
* Base de datos: MySQL
* Arquitectura: API REST (cliente-servidor)

## 🧠 Funcionalidades

* Autenticación de usuarios (registro e inicio de sesión)
* Sistema de colección de cartas
* Apertura de sobres con generación aleatoria de cartas
* Sistema de puntos
* Persistencia de datos en base de datos relacional
* Comunicación entre frontend y backend mediante API

## 🏗️ Arquitectura

La aplicación sigue una arquitectura cliente-servidor:

* El frontend en Angular gestiona la interfaz y las peticiones HTTP
* El backend en Laravel expone una API REST y gestiona la lógica de negocio
* MySQL se encarga del almacenamiento y persistencia de los datos
